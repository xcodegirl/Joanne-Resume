Course Description

This course introduces students to the process of data collection and storage. Students will be introduced to the processes of data collection, integration, storage, cleaning, and data security. Students will also explore the concept of ethical collection, storage and use of data. Students will develop the skills to effectively collect and manage data.

Course Outcomes

Students who successfully complete this course will be able to:

  1. Demonstrate fundamental principles of data collection.

    • Apply the principles of effective data collection.

    • Demonstrate the ability to collect and work with multiple data sources including datasets, cloud-based sources, program interfaces (APIs), web-scraping etc.

    • Demonstrate the ability to collect and work with data in databases, CSV and JSON files.

    • Analyze the potential challenges and limitations of working with multiple data sources.

  1. Apply the fundamental principles of data integration. 

    • Explain the concept and purpose of data integration

    • Summarize data integration methodologies such as ETL (extract, transform, load).

    • Use a variety of data integration tools and techniques (manual integration, middleware data integration, application-based integration, etc.) to create a common data source.

    • Demonstrate how to transport diverse data sources such as File Transfer Protocol (FTP), Application Programming Interfaces (APIs), etc.

    • Create a structured analytical file from raw data.

    • Analyze the benefits and drawbacks of various data integration tools and techniques.

  1. Describe the fundamental principles of data storage.

    • Describe the principles of effective data storage.

    • Describe common considerations related to data storage (location, size, etc.) 

    • Analyze the considerations for data storage based on the organization's specific requirements.

  1. Use the fundamental principles of data cleaning to clean a data set. 

    • Describe the principles which underlie data auditing such as validity, accuracy, completeness, consistency, and uniformity of the data.

    • Explain how to resolve data quality issues (data wrangling, missing values, outliers, unbalanced data, anomalies, data normalization, null values, duplicate removal etc.).

    • Demonstrate how to construct and clean a fully usable dataset.

    • Evaluate different data cleaning techniques and their suitability for different types of data.

  1. Articulate how to ethically collect data.   

    • Describe the ethical issues and legislation (CASL and PIPEDA which impact data collection (privacy protection, governance, security policies, etc.).

    • Summarize the principles of ethical data collection and storage.

  1. Describe the fundamental principles of data security. 

    • Discuss data security breaches across the acquisition, transmission, storage and retrieval phases.

    • Describe a variety of data security controls such as system hardening and monitoring, DBMS configuration, authentication, access, auditing, backups, encryption and application security, etc.

    • Describe how to develop a comprehensive data governance policy for the management of data.

    • Define the main functions of data dictionaries, data catalogs, master data, @and data policies. 

    • Describe how to maintain documented meta-data.

    • Analyze the role and significance of data dictionaries, data catalogs, master data, and data policies in data security.

  1. Describe the basic principles of data governance.

    • Describe the concept of data governance and why it’s important (benefits)

    • Describe the components of data governance (data policies, business policies, data quality, regulatory compliance, master data management, data catalogs, data security)

    • Summarize governance roles and responsibilities at an organization and where the role of a data analyst fits in

    • Explain how to implement data governance

    • Identify data governance best practices and common challenges

    • Summarize the challenges and opportunities in implementing data governance in different organizational settings.
