# T0502 — Object-Oriented Programming (Java)
**Program:** Full Stack Development (FSD)  
**Hours:** 60 hrs (3 weeks)  
**Delivery:** Online Instructor-Guided

---

## Description

Students learn the fundamental principles of object-oriented programming using Java. The course covers class design, encapsulation, inheritance, polymorphism, and abstraction, and introduces collections, exception handling, and file I/O. Students build progressively more complex programs culminating in a multi-class Java application.

---

## Learning Outcomes

**Outcome 1: Apply object-oriented principles to design and implement Java classes**
- Describe the four pillars of object-oriented programming (encapsulation, inheritance, polymorphism, abstraction)
- Define classes with fields, constructors, and methods
- Apply access modifiers to enforce encapsulation
- Instantiate objects and invoke instance methods
- Implement getter and setter methods
- Overload methods and constructors

**Outcome 2: Implement inheritance and polymorphism**
- Create class hierarchies using `extends`
- Override methods and use `super` to access parent class members
- Define and implement interfaces and abstract classes
- Apply polymorphism to write flexible, reusable code
- Use `instanceof` for type checking

**Outcome 3: Build programs using Java collections, exception handling, and file I/O**
- Use `ArrayList`, `HashMap`, and other core collections
- Iterate collections using `for-each` and iterators
- Write try-catch-finally blocks for exception handling
- Create and throw custom exceptions
- Read and write text files using `BufferedReader` / `BufferedWriter`

## Course Schedule

### Module 1 — Classes and Encapsulation (Week 1, 20 hrs)

**Topics:**
- Java project structure and the JDK/IDE setup
- Primitive and reference types
- Class definition: fields, constructors, methods
- Access modifiers: `public`, `private`, `protected`
- Getters, setters, and the JavaBeans convention
- Method and constructor overloading
- `static` vs instance members
- The `toString()` method

**Activities:**
- Define a class from a real-world description
- Write a class with private fields, a constructor, and getters/setters
- Overload constructors to support multiple initialization patterns
- Build a small driver program to instantiate and test the class

---

### Module 2 — Inheritance and Polymorphism (Week 2, 20 hrs)

**Topics:**
- Inheritance: `extends`, constructor chaining, `super`
- Method overriding and `@Override`
- Abstract classes and abstract methods
- Interfaces: `implements`, default methods
- Polymorphism: runtime dispatch, type casting
- `instanceof` and safe casting

**Activities:**
- Build a class hierarchy (e.g., Vehicle → Car, Truck)
- Demonstrate polymorphism by storing subtype objects in a parent-type array
- Define an interface and implement it in multiple classes
- Compare interface vs abstract class design decisions

---

### Module 3 — Collections, Exceptions, and File I/O (Week 3, 20 hrs)

**Topics:**
- `ArrayList` and `HashMap`: creation, access, iteration
- Generics syntax and why it matters
- Exception hierarchy: checked vs unchecked
- `try-catch-finally`; multiple `catch` blocks
- Custom exception classes
- Reading and writing text files
- Combining: persist and load a collection to/from a file

**Activities:**
- Store objects in an `ArrayList`; search and filter with loops
- Use a `HashMap` as a lookup structure
- Wrap risky operations in try-catch; create a custom exception
- Read a CSV file and parse records into objects
- Write the object collection back to file; verify round-trip
